If you’re using **Cloudflare Workers**, store sensitive values as **Secrets** in the Cloudflare dashboard, not in code or plain environment variables. Cloudflare treats secrets like environment variables, but their values are hidden after creation in the dashboard and Wrangler, which is why API keys and tokens should go there.[1][2]

## In dashboard

1. Open the [Cloudflare dashboard](https://dash.cloudflare.com/) and go to **Workers & Pages**.[2][1]
2. Select your Worker, then open **Settings**.[1][2]
3. Find **Variables and Secrets** and click **Add**.[2][1]
4. Choose **Secret** as the type, then enter:
- Variable name, for example `STRIPE_SECRET_KEY`
- Value, for example your actual Stripe secret key[2]
5. Click **Deploy** or **Save** to apply the change.[1][2]

## What to store

Use **Secrets** for anything sensitive, such as:

- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`
- `TURNSTILE_SECRET`
- API tokens, database passwords, or private signing keys[3][4]

Use normal environment variables only for non-sensitive values, such as public URLs or public site keys.[5][1]

## In code

Inside a Worker, you access a secret through the `env` object, the same way you access any other binding. For example, a secret named `STRIPE_SECRET_KEY` is available as `env.STRIPE_SECRET_KEY` in your Worker code.[2]

Example:

```js
export default {
  async fetch(request, env) {
    const stripeKey = env.STRIPE_SECRET_KEY;
    return new Response("OK");
  }
}
```

## Account-level option

Cloudflare also has **Secrets Store**, which is a centralized account-level secret manager that you can bind to Workers if you need to reuse secrets across multiple Workers. In the dashboard, you create a secret in **Secrets Store**, then add a binding from your Worker under **Settings → Bindings** so the Worker can use it.[6]

## Best practice

Do not hardcode secrets in source files or commit them to Git. Keep public values like `TURNSTILE_SITE_KEY` as regular variables if needed, but store private values like `TURNSTILE_SECRET` and Stripe secrets as encrypted Cloudflare secrets instead.[7][1]
