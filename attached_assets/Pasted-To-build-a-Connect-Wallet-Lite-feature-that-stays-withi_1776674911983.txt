To build a **"Connect Wallet" Lite** feature that stays within your Jekyll and Cloudflare stack, you need a workflow that avoids heavy npm packages (like Wagmi or RainbowKit) in favor of lightweight, browser-native interactions.

The goal is to have a user connect their wallet, send their address to a **Cloudflare Worker**, and receive a "Risk Audit" based on the protocols they are currently interacting with.

### 1. The "Lite" Logic (Exposure Detection)
Since you want to keep it high-value but "Lite," the logic should be:
* **Step 1:** Use `window.ethereum` to get the user's address.
* **Step 2:** Use the **Etherscan API** (Account/TokenList) to see which tokens/contracts the user holds.
* **Step 3:** Cross-reference those contracts with your **DeFiScoring Database** (managed by your Worker).
* **Step 4:** Display only the "Low Score" protocols the user is currently "exposed" to.

> **Task 4: UI Feedback**

### 3. Implementation Flow in your Jekyll Site

To make this work on GitHub Pages, you will add the generated code in three specific places:

| Component | File Location | Purpose |
| :--- | :--- | :--- |
| **The Button** | `_includes/wallet-check.html` | The "Connect Wallet" button and the results table. |
| **The Logic** | `assets/js/risk-scanner.js` | The JS that talks to the Cloudflare Worker. |
| **The Data** | `_data/protocols.yml` | Your "Source of Truth" list of contract addresses and their scores. |
