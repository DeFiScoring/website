### 2. Step-by-Step Setup (Do This Today)

1. **GitHub Repo**
   - Create repo: `de-fi-scoring`
   - Add Jekyll: `jekyll new . --force` (or use a clean theme like Minimal or Chirpy for dashboard look).
   - Folder structure:
     ```
     /_site/          ← built static files
     /_layouts/
     /assets/js/      ← wallet-connect.js, charts.js
     /api/            ← (optional) if using Pages Functions
     wrangler.toml    ← for Workers
     ```

2. **Cloudflare Pages (Jekyll Hosting)**
   - Go to Cloudflare → Workers & Pages → Create → Pages → Connect GitHub repo.
   - Framework preset: **Jekyll**
   - Build command: `jekyll build`
   - Build output directory: `_site`
   - Done. Your site will be at `https://your-project.pages.dev` (add custom domain later).

3. **Cloudflare Workers (Dynamic Backend)**
   - Install Wrangler: `npm install -g wrangler`
   - `wrangler init scoring-api` (or put in same repo).
   - Bindings you’ll create:
     - D1 database (`de-fi-scores`)
     - KV namespace (`SCORE_CACHE`)
     - R2 bucket (`reports`)
   - Deploy: `wrangler deploy`

4. **Gmail Emails (Notifications)**
   - Enable 2FA on your Gmail.
   - Generate App Password (16 chars).
   - In Worker code, use SMTP (simple `fetch` + nodemailer polyfill or native SMTP via Cloudflare’s TCP support):

### 4. Limitations & How We Handle Them (Realistically)

- **No heavy ML**: Rule-based + heuristics first. Later you can call free external APIs (e.g. Hugging Face inference) from the Worker if needed.
- **RPC rate limits**: Cache aggressively in KV/D1. Use multiple free RPC providers.
- **Data freshness**: Worker can re-scan on demand (user-initiated).
- **Scale**: Free tier is generous for MVP (D1 = 5M rows/day read, KV = millions of reads). Paid is still cheap.
